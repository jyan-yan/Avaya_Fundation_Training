**GCC (GNU Compiler Collection)** is a powerful and widely used tool in the software world, especially on Linux and Unix systems. Its primary job is to act as a translator. Computers only understand binary (0s and 1s), but humans write code in high-level languages like C or C++. GCC takes the human-readable source code you write and translates it into the machine code that the computer's processor can execute.

# The Four Steps of GCC

When you run a simple command like `gcc main.c`, GCC does not do the translation in one single jump. It actually performs four distinct steps behind the scenes.

1. **Preprocessor:** This step prepares your code. It removes all your comments, expands macros (like `#define`), and inserts the actual contents of header files (like `#include <stdio.h>`) directly into your source code.
2. **Compiler:** The compiler takes the preprocessed code and translates it into **Assembly Language**. This is a lower-level language that represents the logic of your code in instructions that are closer to hardware.
3. **Assembler:** The assembler converts the assembly code into **Machine Code** (binary instructions of 0s and 1s) that the computer's CPU can actually understand. The output is an intermediate file called an Object file.
4. **Linker:** Most programs rely on external resources (like the `printf` function from the C standard library). The linker combines your object file with these required external libraries to produce the final, ready-to-run executable file.

# From ELF to Process

In Linux, the final executable file generated by the Linker is called an **ELF** (Executable and Linkable Format).

It is important to understand the difference between a file and a process. An ELF file is just a static bundle of data resting on your hard drive. It does nothing on its own. However, the moment you tell Linux to run this ELF file, the operating system loads it into the computer's active memory (RAM). When the ELF file "wakes up" and starts executing, it becomes a **Process**. A process is simply a program in action.

# Verification

Let's write a quick program and use simple Linux commands to see these concepts in reality.

1. **Create a simple C file** 

Open your terminal and create a file named `loop.c`:

```C
#define LOOP 1 

int main(void) 
{
	while(LOOP); 
}
```

2. **See the GCC Steps** 

Run these commands to see the output of each translation step:

- `gcc -E loop.c -o loop.i` (Preprocess: creates a huge text file).
- `gcc -S loop.i -o loop.s` (Compile: creates Assembly code).
- `gcc -c loop.s -o loop.o` (Assemble: creates the binary Object file).
- `gcc loop.o -o loop`(Link: creates the final ELF executable).

3. **Run and Manage the Process**

Because this code is an infinite loop, we will run it in the background so it doesn't freeze our terminal.

- **Start the process**: Run `./loop &`. The & symbol tells Linux to run it in the background. The terminal will reply with a Process ID (PID), looking something like this: [1] 12345. Note that number!
- **Monitor the process**: Run `top -p 12345` (replace 12345 with your actual PID). This opens a live task manager. You will see your loop process actively running and consuming CPU. Press q to exit this view.
- **Kill the process**: Since it is an infinite loop, you must stop it manually. Run `kill 12345` (using your PID) to terminate the process and free up your system's memory.
